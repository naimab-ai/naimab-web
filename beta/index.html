What Testers Get

A beta page that feels like part of the product, not an afterthought

The download flow keeps the same Naimab tone and visual language while giving you a practical place to ship desktop builds to testers.

01

One stable link

Testers can always use the same `/testx/download` URL. You replace the installer once, and every CTA on the page starts serving the new build.

  • No link rotation for each release
  • Works as a direct attachment download
  • Friendly fallback when the installer is missing
02

On-brand presentation

The page reuses the same typography, colors, grain, CTA treatment, and overall rhythm as the landing so it feels native to the current site.

  • Shared Halant and Geist typography
  • Same warm palette and pill buttons
  • Dashboard-led product framing
03

Tester-friendly status

The page checks whether a build is actually live and updates the UI accordingly, so you avoid sending people into a dead download flow.

  • Ready, missing, and error states
  • Windows-aware helper copy
  • Automatic file size display when available
How It Works

A simple release flow for each new Windows beta

01

Drop in the new installer

Place the latest Windows build in the repo at the expected download path and deploy the site.

02

Keep the public link

The page and direct download endpoint stay the same, so you do not need to update the beta CTA or resend a new URL.

03

Let testers pull the latest build

Once deployed, the button becomes live and the page can surface the download size automatically.
